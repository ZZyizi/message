# 一对一聊天端到端加密 (E2EE) 设计

## 概述

为一对一聊天实现端到端加密，保护消息在传输过程中不被 Relay 服务器窃听。采用 X25519 ECDH 密钥协商 + AES-256-GCM 对称加密方案，提供前向保密。

## 目标

- 消息 payload 在传输和存储中均为密文
- Relay 服务器无法读取消息内容
- 每次会话使用临时密钥，提供前向保密
- 复用已有的密码学原语（`derive_session_key`、`encrypt_message`、`decrypt_message`）

## 非目标

- 群组加密（后续实现）
- 多设备密钥同步（后续实现）
- Double Ratchet / Signal 协议（当前方案已满足安全需求）

## 密钥协商流程

```
Alice                                    Relay                                   Bob
  |                                        |                                      |
  |-- KeyExchange {ephemeral_pubkey} ----->|                                      |
  |                                        |-- KeyExchange {ephemeral_pubkey} --->|
  |                                        |                                      |
  |<-- KeyExchange {ephemeral_pubkey} -----|<-- KeyExchange {ephemeral_pubkey} ---|
  |                                        |                                      |
  | SessionKey = ECDH(A_priv, B_pub)      |        SessionKey = ECDH(B_priv, A_pub)
  |                                        |                                      |
  |== Encrypted Message (AES-256-GCM) ===> Relay ===> Bob decrypts ==============|
```

- 临时公钥通过 Relay 明文交换（不含私钥，不影响安全）
- 双方各自计算出相同的 SessionKey
- SessionKey 仅存内存，不写入数据库
- 每次打开聊天窗口时重新协商

## WebSocket 协议扩展

在 `WsMessage` 枚举中新增变体：

```rust
KeyExchange {
    from: String,              // 发送方静态公钥
    ephemeral_pubkey: String,  // 临时 X25519 公钥 (Base64)
}
```

## 消息加解密

### 发送路径

```
明文 payload
  -> encrypt_message(payload, session_key, None)
  -> 密文 (Base64)
  -> 存入 messages.payload + pending_messages.payload
  -> 通过 WebSocket 发送到 Relay
```

### 接收路径

```
从数据库/Relay 收到密文
  -> decrypt_message(ciphertext, session_key)
  -> 明文
  -> 通过 Tauri 事件发送到前端显示
```

### 数据库

`messages.payload` 存储密文（Base64 编码），现有表结构无需修改。

## 会话状态管理

```rust
struct SessionState {
    peer_static_pubkey: String,
    my_ephemeral_privkey: [u8; 32],
    peer_ephemeral_pubkey: Option<[u8; 32]>,
    session_key: Option<[u8; 32]>,
}
```

存储在 `AppState` 中：

```rust
pub struct AppState {
    pub db: Arc<Mutex<db::Database>>,
    pub identity: Arc<RwLock<identity::IdentityManager>>,
    pub sessions: Arc<RwLock<HashMap<String, SessionState>>>,  // 新增
}
```

### 生命周期

1. 打开聊天窗口 → 生成临时密钥对 → `session_key = None`
2. 收到对方 `KeyExchange` → `derive_session_key()` → `session_key = Some(key)`
3. 发送/接收消息时检查 `session_key.is_some()`
4. 关闭聊天窗口 → **销毁 SessionState**（清零内存）

## 边界情况

### 对方离线

- 临时公钥已发出，对方未回复 KeyExchange
- 消息存入 pending_messages，标记"等待密钥协商"
- 对方上线后收到 KeyExchange → 派生 SessionKey → 自动重发 pending 消息

### 网络断开

- 重新连接后发起新 KeyExchange，协商新 SessionKey
- 旧的未确认消息用新密钥重新加密后重发

### 前端交互

- 打开聊天窗口时自动发起 KeyExchange
- SessionKey 未就绪时：发送按钮禁用，显示"正在建立安全通道..."
- 收到对方临时公钥后：输入框启用

## 改动范围

| 文件 | 改动 |
|---|---|
| `relay.rs` | 新增 `KeyExchange` 消息处理 + 密钥协商逻辑 |
| `message.rs` | 发送/接收路径接入加密/解密 |
| `lib.rs` | AppState 新增 `sessions` 字段 |
| `crypto.rs` | 无新函数，复用已有原语 |
| `db.rs` | 无表结构改动 |
| 前端 `+page.svelte` | 打开聊天时触发协商，SessionKey 就绪前禁用输入 |

## 复用已有函数

- `crypto::derive_session_key()` — X25519 ECDH + HKDF 派生会话密钥
- `crypto::encrypt_message()` — AES-256-GCM 加密
- `crypto::decrypt_message()` — AES-256-GCM 解密
- `crypto::generate_device_keypair()` — 生成临时 X25519 密钥对
