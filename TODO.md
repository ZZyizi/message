  - 配置 Svelte 前端框架

## 阶段三：消息系统

- [ ] **T3.1** 实现 Relay WebSocket 客户端
  - WebSocket 连接管理
  - 心跳机制（30s）
  - 重连策略（指数退避）
  - 在线状态管理

- [ ] **T3.2** 实现消息 ACK 机制与 Pending Queue
  - 发送消息后加入 pending queue
  - 收到 ACK 后移除
  - 未确认消息超时处理
  - 重发机制

- [ ] **T3.3** 实现离线消息补发
  - 分页拉取 pending 消息
  - seq_id 排序
  - 冲突处理（保留较早消息）
  - 群主 pubkey 优先级排序
  - 过期消息标记（7 天窗口）

- [ ] **T3.4** 实现消息撤回功能
  - 撤回事件发送
  - 已上线设备立即广播
  - 离线设备同步时处理撤回
  - 本地消息删除（保留撤回标记）

  - 加入群组（获取群密钥）

## 阶段五：多设备系统

- [ ] **T5.1** 实现多设备绑定流程
  - 主设备生成授权 token（UUID + 5 分钟有效期）
  - token 二维码生成（含 token + 主设备公钥）
  - 新设备扫码
  - 新设备生成 Device Key
  - 请求主设备授权
  - 主设备签名返回授权证书
  - 新设备保存授权证书

- [ ] **T5.2** 实现设备撤销功能
  - 主设备发起撤销请求
  - Relay 更新黑名单
  - 被撤销设备连接被拒绝（403）
  - 广播设备变更事件
